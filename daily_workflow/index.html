Flujo de trabajo diario

Aquí encontrarás toda la información que se considera necesaria para ser productivo en el dia a día del proyecto, empezando por el algoritmo que todo miembro del equipo debe seguir:

  1. Debes estar en la rama develop: git switch develop
  2. Crea una rama a partir de develop: git switch -c <name>. Reemplaza <name> con el nombre de la rama.
  3. Lleva a cabo tus cambios localmente.
  4. Crea un commit.
  5. Haz push de tu rama al repositorio remoto.
  6. Crea una pull request desde tu rama hacia develop.
  7. Si el sistema de integración continua falla al procesar tus cambios, revisa qué ocurrió y regresa al paso 3, de forma que los cambios locales que hagas arreglen los problemas causantes de la falla.
  8. Espera a que alguien haga una code review de tu código e integre tus cambios a develop.
  9. Regresa al paso 1.

Ejecutar el backend/frontend

Puedes ejecutar los servidores de frontend y backend de la siguiente forma:

  • Solo backend (Django): task backend
  • Solo frontend (React): task frontend
  • Backend y frontend: task dev

Probar JWT

La forma más rápida de probar que JWT retorne tokens para un determinado usuario es con el siguiente comando dependiendo del sistema operativo:

  • Unix:
curl \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{"username": "<username>", "password": "<password>"}' \
  http://localhost:8000/api/login/
  • Windows:
curl.exe `
  -X POST `
  -H "Content-Type: application/json" `
  -d "{\"username\":\"<username>\",\"password\":\"<password>\"}" `
  "http://127.0.0.1:8000/api/login/"

Debes reemplazar <username> y <password> con el nombre de usuario y contraseña, respectivamente.
